== 介紹

比特幣((("比特幣", "運作概述", id="bitcoin-operational-overview")))是一組概念和技術，構成了數字貨幣生態系統的基礎。在比特幣網絡中的參與者之間，用於儲存和傳輸價值的貨幣單位被稱為比特幣。比特幣用戶主要通過互聯網使用比特幣協議進行交流，儘管也可以使用其他傳輸網絡。比特幣協議棧作為開源軟體，可以在各種計算設備上運行，包括筆記本電腦和智能手機，使該技術容易獲取。

[TIP]
====
在本書中，貨幣單位被稱為 "bitcoin"，以小寫 _b_ 表示，
而整個系統稱為 "Bitcoin"，以大寫 _B_ 表示。
====

用戶可以通過網絡轉移比特幣，用於與傳統貨幣幾乎相同的用途，包括購買和銷售商品、向個人或組織匯款，或延長信用。比特幣可以在專門的貨幣交易所購買、出售和兌換其他貨幣。可以說，比特幣是互聯網上最完美的貨幣形式，因為它快速、安全且無國界。

與傳統貨幣不同，比特幣完全是虛擬的。沒有實體硬幣，也沒有獨立的數字硬幣。這些硬幣在交易中以價值從支付方轉移到接收方的形式被隱含存在。比特幣用戶控制密鑰，這些密鑰可以讓他們在比特幣網絡中證明對比特幣的擁有權。使用這些密鑰，他們可以簽署交易來解鎖價值，並通過將其轉移給新擁有者來消費比特幣。密鑰通常存儲在每個用戶計算機或智能手機的數字錢包中。擁有能簽署交易的密鑰是唯一花費比特幣的前提條件，這將控制權完全放在每個用戶手中。

比特幣是一個分佈式的點對點系統。因此，沒有中央伺服器或控制點。比特幣單位是通過稱為「挖礦」((("比特幣", "挖礦")))("挖礦")))的過程創建的，該過程涉及反覆執行引用最近比特幣交易列表的計算任務。任何比特幣網絡的參與者都可以作為礦工運行，使用他們的計算設備來幫助保護交易。平均每 10 分鐘，就有一位比特幣礦工可以為過去的交易添加安全性，並獲得全新的比特幣和最近交易支付的手續費作為獎勵。本質上，比特幣挖礦去中心化了中央銀行的貨幣發行和結算功能，取代了對任何中央銀行的需求。

比特幣協議包括內建的算法，用於在整個網絡中調節挖礦功能。礦工必須執行的計算任務的難度會動態調整，以確保無論有多少礦工（以及有多少處理能力）競爭，平均每 10 分鐘就有一人成功。此外，協議也會定期減少新比特幣的創造數量，將總共將創造的比特幣數量限制在 2100 萬枚以下。結果是，流通中的比特幣數量緊隨一條可以輕鬆預測的曲線，每四年會有剩餘硬幣的一半進入流通。在大約第 1,411,200 個區塊（預計在 2035 年產生）時，將發行 99% 的所有比特幣。由於比特幣的減發率，長期來看，比特幣是一種通縮貨幣。此外，任何人都無法強迫你接受任何超出預期發行率創建的比特幣。

在幕後，比特幣也是協議、點對點網絡和分佈式計算創新技術的名稱。比特幣建立在密碼學和分佈式系統的幾十年研究基礎上，並將至少四個關鍵創新結合在一起，形成獨特而強大的組合。比特幣由以下組成：

* 去中心化的點對點網絡（比特幣協議）
* 公共交易日誌（區塊鏈）
* 用於獨立交易驗證和貨幣發行的規則集（共識規則）
* 用於達成全球去中心化共識的機制（工作量證明算法）

作為開發者，我認為比特幣類似於價值互聯網，一個用於傳播價值並通過分佈式計算保護數字資產所有權的網絡。比特幣的內容遠不止乍看((("比特幣", "運作概述", startref="bitcoin-operational-overview")))所見。

在本章中，我們將從解釋一些主要概念和術語、獲取必要的軟體以及使用比特幣進行簡單交易開始。在接下來的章節中，我們將開始剖析構成比特幣的技術層，並檢查比特幣網絡和協議的內部運作。

.比特幣之前的數字貨幣
****

((("數字貨幣", "歷史")))("歷史", "數字貨幣的歷史", secondary-sortas="數字貨幣"))(("密碼學"))可行的數字貨幣的出現與密碼學的發展息息相關。當我們考慮使用位元來表示可以用來交換商品和服務的價值所涉及的基本挑戰時，這一點就不足為奇了。任何接受數字貨幣的人必須解決的三個基本問題是：

* 我能相信這些貨幣是真實而非偽造的嗎？
* 我能相信這些數字貨幣只能被花費一次（即「雙重花費」問題）嗎？
* 我能確定沒有人能聲稱這些貨幣屬於他們而不是我嗎？

紙幣的發行者通過使用越來越複雜的紙張和印刷技術來不斷打擊偽造問題。實體貨幣很容易解決雙重花費問題，因為同一紙幣不可能同時出現在兩個地方。當然，傳統貨幣也經常以數字形式儲存和傳輸。在這些情況下，偽造和雙重花費問題通過中央權威機構對所有電子交易進行清算處理，從而獲得解決。對於無法利用特殊油墨或全息條帶的數字貨幣，密碼學提供了信任用戶對價值主張的合法性的基礎。具體來說，密碼學的數字簽名使用戶可以簽署數字資產或交易，以證明對該資產的所有權。通過適當的架構，數字簽名也可以用來解決雙重花費問題。

當密碼學在 1980 年代後期變得更加廣泛可用和理解時，許多研究人員開始嘗試使用密碼學來建立數字貨幣。這些早期的數字貨幣項目發行了數字貨幣，通常由國家貨幣或貴金屬（如黃金）支持。

儘管這些早期的數字貨幣運作良好，但它們是中心化的，因此很容易受到政府和黑客的攻擊。早期的數字貨幣使用中央結算所來定期結算所有交易，就像傳統的銀行系統一樣。不幸的是，在大多數情況下，這些新生的數字貨幣成為了擔憂的政府的目標，最終被訴訟至消失。有些則在母公司突然清算時發生了戲劇性的崩潰。為了對抗合法政府或犯罪勢力等敵對勢力的干涉，需要一種去中心化的數字貨幣來避免單點攻擊。比特幣就是這樣一個系統，從設計上就是去中心化的，不受任何中央權威或控制點的影響，也不容易被攻擊或腐敗。

****

=== 比特幣的歷史

比特幣((("比特幣", "歷史")))("歷史", "比特幣的歷史", secondary-sortas="比特幣"))(("中本聰"))最早於 2008 年由一篇名為《比特幣：一個點對點的電子現金系統》footnote:[https://oreil.ly/KUaBM[《比特幣：一個點對點的電子現金系統》, 中本聰。]]的論文中被描述。這篇論文是由化名中本聰發表的（見 <<satoshi_whitepaper>>）。中本聰結合了數字簽名和 Hashcash 等多個先前的發明，創建了一個完全去中心化的電子現金系統，該系統不依賴於中央機構進行貨幣發行或交易的結算與驗證。一項關鍵的創新是使用一個分佈式計算系統（稱為「工作量證明」算法）來每 10 分鐘平均進行一次全球性抽籤，讓去中心化網絡對交易狀態達成_共識_。這優雅地解決了雙重花費問題，即單一貨幣單位可能被花費兩次的問題。此前，雙重花費問題一直是數字貨幣的弱點，需通過中央結算所來處理所有交易才能解決。

比特幣網絡於 2009 年啟動，基於中本聰發佈的參考實現，並且由許多其他程序員不斷修訂和改進。運行工作量證明算法（挖礦）((("工作量證明算法", "另見挖礦")))的機器數量和算力大幅增長，這些算力為比特幣提供了安全性和彈性，並且現在的綜合計算能力已經超過了世界頂級超級計算機的總計算操作次數。

中本聰於 2011 年 4 月退出公眾視野，將開發代碼和網絡的責任交給了一個充滿活力的志願者群體。至今，比特幣背後的個人或群體的身份仍然未知。然而，無論是中本聰還是其他任何人，都無法對比特幣系統施加個人控制。比特幣系統的運行基於完全透明的數學原理、開源代碼，以及參與者之間的共識。這項發明本身是突破性的，並且已經在分佈式計算、經濟學和計量經濟學等領域催生了新的科學。

=== 入門

比特幣((("比特幣", "錢包", see="錢包")))("錢包", "解釋")))是一種可以通過使用支持該協議的應用程序來訪問的協議。「比特幣錢包」是比特幣系統最常見的用戶界面，就像網頁瀏覽器是 HTTP 協議最常見的用戶界面一樣。有很多不同實現和品牌的比特幣錢包，就像有很多品牌的網頁瀏覽器（例如 Chrome、Safari 和 Firefox）。而且就像我們都有自己最喜愛的瀏覽器一樣，比特幣錢包在質量、性能、安全性、隱私和可靠性上也各不相同。還有一個比特幣協議的參考實現，其中包含了一個錢包，稱為「Bitcoin Core」，它源自中本聰所寫的原始實現。

==== 選擇比特幣錢包

比特幣錢包((("錢包", "選擇")))("選擇", "錢包"))((("選擇", see="選擇")))是比特幣生態系統中開發最活躍的應用之一。競爭非常激烈，雖然可能現在正在開發新的錢包，但去年的一些錢包已不再積極維護。許多錢包專注於特定的平台或用途，有些更適合初學者，而另一些則充滿了適合進階用戶的功能。選擇錢包是高度主觀的，取決於用途和用戶的專業知識。因此，推薦特定品牌或錢包是沒有意義的。然而，我們可以根據其平台和功能對比特幣錢包進行分類，並對現有的不同類型的錢包提供一些清晰的解釋。值得嘗試幾個不同的錢包，直到找到適合自己需求的。

===== 比特幣錢包的類型
比特幣錢包((("錢包", "類型", id="wallet-type")))可以根據平台分類如下：

桌面錢包:: 桌面錢包((("桌面錢包")))是第一種類型的比特幣錢包，被創建為參考實現。許多用戶運行桌面錢包是因為它們提供的功能、自主性和控制力。然而，運行在 Windows 和 macOS 等通用操作系統上具有某些安全劣勢，因為這些平台通常不安全且配置不佳。

移動錢包:: 移動錢包((("移動錢包")))是最常見的比特幣錢包類型。這些錢包運行在 Apple iOS 和 Android 等智能手機操作系統上，通常是新用戶的絕佳選擇。許多移動錢包設計簡單易用，但也有針對進階用戶的全功能移動錢包。為了避免下載和存儲大量數據，大多數移動錢包從遠程伺服器檢索信息，這可能會通過向第三方披露比特幣地址和餘額信息而降低您的隱私。

網頁錢包:: 網頁錢包((("網頁錢包")))通過網頁瀏覽器訪問，用戶的錢包存儲在第三方擁有的伺服器上。這類似於網頁郵件，完全依賴於第三方伺服器運行。一些這樣的服務在用戶的瀏覽器中運行客戶端代碼，這使得比特幣密鑰的控制權仍在用戶手中，儘管用戶對伺服器的依賴仍然會影響隱私。然而，大多數服務為了易用性而控制用戶的比特幣密鑰。不建議在第三方系統上存儲大量比特幣。

硬件簽名設備:: 硬件簽名設備((("硬件簽名設備")))是可以使用專用硬件和韌體存儲密鑰並簽署交易的設備。它們通常通過 USB 線、近場通訊 (NFC) 或帶有 QR 碼的相機連接到桌面、移動或網頁錢包。通過在專用硬件上處理所有與比特幣相關的操作，這些錢包減少了對許多類型攻擊的脆弱性。硬件簽名設備有時被稱為「硬件錢包」，但它們需要與全功能錢包配對才能發送和接收交易，而這些配對錢包提供的安全性和隱私在用戶使用硬件簽名設備時的安全性和隱私中扮演著關鍵角色。

===== 全節點與輕量客戶端
另一種對比特幣錢包進行分類的方法是根據它們的自主程度以及它們如何與比特幣網絡進行交互：

全節點:: 全節點((("全節點")))是一個驗證比特幣交易的完整歷史（每個用戶的每一筆交易）的程序。可選地，全節點還可以存儲先前已驗證的交易，並向其他比特幣程序提供數據，不論是同一台計算機上還是通過互聯網。一個全節點使用了大量計算機資源——大約每天處理比特幣交易所需的計算資源相當於觀看一小時的流媒體視頻，但全節點為其用戶提供完全的自主性。

輕量客戶端:: 輕量客戶端((("輕量客戶端")))((("簡化支付驗證 (SPV) 客戶端")))("SPV (簡化支付驗證) 客戶端")))也被稱為簡化支付驗證 (SPV) 客戶端，通過接收和發送比特幣交易信息連接到全節點或其他遠程伺服器，但本地存儲用戶錢包，對接收到的交易進行部分驗證，並獨立創建外發交易。

第三方 API 客戶端:: 第三方 API 客戶端((("第三方 API 客戶端")))是指通過第三方 API 系統而不是直接連接到比特幣網絡與比特幣進行交互的客戶端。錢包可能由用戶存儲，也可能由第三方伺服器存儲，但該客戶端信任遠程伺服器為其提供準確的信息並保護其((("錢包", "類型", startref="wallet-type")))隱私。

[TIP]
====
比特幣((("比特幣", "作為點對點網絡", secondary-sortas="點對點網絡")))("點對點網絡，比特幣作為")))((("點對點")))((("客戶端")))是一個點對點（P2P）網絡。全節點是_點_: 每個點都能獨立驗證每筆確認的交易，並能為其用戶提供完全權威的數據。輕量錢包和其他軟件是_客戶端_: 每個客戶端依賴於一個或多個點為其提供有效的數據。比特幣客戶端可以對其接收到的部分數據進行次級驗證，並與多個點建立連接，以減少對單一點完整性的依賴，但最終客戶端的安全依賴於其點的完整性。
====

===== 誰控制密鑰

一個非常((("錢包", "密鑰控制")))("密鑰", "密鑰控制"))((("比特幣", "密鑰控制")))重要的額外考量因素是_誰控制密鑰_。正如我們在後續章節中將看到的，訪問比特幣是通過「私鑰」控制的，這些私鑰類似於非常長的 PIN 碼。如果只有你能控制這些私鑰，那麼你就掌控了你的比特幣。相反，如果你無法控制這些私鑰，那麼你的比特幣就是由第三方管理的，最終由他們代你控制你的資金。密鑰管理軟體根據控制權分為兩個重要類別：_錢包_，由你自己控制密鑰，和由託管人控制密鑰的資金和賬戶。為了強調這一點，我（Andreas）創造了這樣一句話：_你的密鑰，你的幣。不掌握密鑰，就不擁有幣_。

將這些分類結合起來，許多比特幣錢包可以分為幾組，最常見的三種是桌面全節點（你控制密鑰）、移動輕量錢包（你控制密鑰）、以及第三方的網頁賬戶（你不控制密鑰）。不同類別之間有時存在模糊界限，因為軟體在多個平台上運行，並可以以不同方式與網絡互動。

愛麗絲不是技術用戶，最近才從她的朋友喬那裡聽說比特幣。在一次派對上，喬熱情地向周圍的每個人解釋比特幣，並提供了一次演示。愛麗絲被吸引，問她如何開始使用比特幣。喬說移動錢包最適合新用戶，並推薦了幾個他最喜歡的錢包。愛麗絲下載了喬推薦的其中一個錢包並安裝在她的手機上。

當愛麗絲第一次運行她的錢包應用程序時，她選擇了創建一個新的比特幣錢包的選項。由於她選擇的錢包是((("錢包", "非託管")))("非託管錢包")))非託管錢包，所以愛麗絲（且只有愛麗絲）將控制她的密鑰。因此，她有責任備份這些密鑰，因為一旦丟失密鑰，她將無法訪問她的比特幣。為了方便她的備份，她的錢包生成了一個_恢復碼_，可用於恢復她的錢包。

[TIP]
====
恢復碼((("助記詞", see="恢復碼")))("種子詞", see="恢復碼")))有時被稱為「助記詞」或「助記短語」，這意味著你應該記住這些詞組，但將這些詞寫在紙上所需的工作量較少，且通常比大多數人的記憶更可靠。另一個替代名稱是「種子短語」，因為它為生成錢包所有密鑰的函數提供了輸入（「種子」）。
====

如果愛麗絲的錢包出現問題，她可以下載一個新版本的錢包軟體，並輸入這個恢復碼來重建她發送或接收的所有鏈上交易的錢包資料庫。然而，僅靠恢復碼恢復並不會自動恢復愛麗絲在錢包中輸入的任何其他數據，例如她與特定地址或交易關聯的標籤。雖然失去這些元數據的影響不如失去比特幣那麼重要，但它依然可能有其重要性。想像一下，你需要查看一份舊的銀行或信用卡對帳單，但每個付款或收款方的名稱都被抹掉了。為了防止丟失元數據，許多錢包提供了恢復碼之外的額外備份功能。

對於某些錢包來說，這種額外的備份功能比以往任何時候都更重要。現在許多比特幣支付((("鏈下技術")))都是使用_鏈下_技術進行的，其中並非每一筆支付都存儲在公共區塊鏈中。這減少了用戶的成本並改善了隱私，這些都是其優勢，但這也意味著依賴鏈上數據的恢復碼無法保證恢復所有用戶的比特幣。對於支持鏈下功能的應用程序，定期備份錢包資料庫非常重要。

值得注意的是，在第一次向新移動錢包接收資金時，許多錢包通常會重新驗證你是否已經安全地備份了恢復碼。這可能只是簡單的提示，也可能要求用戶手動重新輸入恢復碼。

[WARNING]
====
雖然許多合法的錢包會提示你重新輸入恢復碼，但也有很多惡意軟體模仿錢包的設計，要求你輸入恢復碼，然後將輸入的恢復碼轉發給惡意軟體開發者，以便他們竊取你的資金。這類行為類似於釣魚網站，試圖騙取你的銀行密碼。對於大多數錢包應用程序來說，它們只會在初次設置（在你收到任何比特幣之前）和恢復（在你失去對原始錢包的訪問權限之後）時要求輸入恢復碼。如果應用程序在其他時間要求你的恢復碼，請諮詢專家以確保你沒有((("錢包", "恢復碼", startref="wallet-recovery")))("恢復碼", startref="recovery-code")))被釣魚攻擊。
====

==== 比特幣地址

愛麗絲現在((("地址", "解釋")))準備好開始使用她的新比特幣錢包了。她的錢包應用程序隨機生成了一個私鑰（詳細描述見 <<private_keys>>），該私鑰將用於導出指向她錢包的比特幣地址。目前，她的比特幣地址對比特幣網絡來說是未知的，也沒有在比特幣系統的任何部分「註冊」。她的比特幣地址只是與她的私鑰相對應的數字，她可以用它來控制對資金的訪問。這些地址由她的錢包獨立生成，不需要參考或註冊任何服務。

[TIP]
====
比特幣地址和發票有多種格式。((("發票")))地址和發票可以與其他比特幣用戶共享，對方可以使用它們將比特幣直接發送到你的錢包。你可以將地址或發票分享給他人，而不必擔心你的比特幣的安全性。與銀行賬號不同，任何知道你的比特幣地址的人都無法從你的錢包中取款——你必須自己發起所有的支付。然而，如果你給兩個人相同的地址，他們將能看到對方給你發送了多少比特幣。如果你公開發佈你的地址，所有人都將能看到其他人發送到該地址的比特幣數量。為了保護你的隱私，你應該每次請求付款時生成一個帶有新地址的新發票。
====

[[getting_first_bitcoin]]
==== 獲取你的第一個比特幣

新用戶的第一個任務是((("比特幣", "獲取", id="bitcoin-acquire")))("獲取比特幣", id="acquire-bitcoin")))獲取一些比特幣。

比特幣交易是不可逆的。大多數電子支付網絡（例如信用卡、借記卡、PayPal 和銀行賬戶轉賬）都是可逆的。對於賣出比特幣的人來說，這一差異引入了非常高的風險，買家在收到比特幣後可能會撤銷電子支付，從而詐騙賣家。為了減少這種風險，以傳統電子支付換取比特幣的公司通常要求買家進行身份驗證和信用評估，這可能需要幾天或幾周的時間。對於新用戶來說，這意味著你無法立即用信用卡購買比特幣。然而，只要有一些耐心和創造性的思考，你並不需要這麼做。

以下是新用戶獲取比特幣的一些方法：

* 找一個擁有比特幣的朋友，直接從他或她那裡購買。許多比特幣用戶都是這樣開始的。這種方法最為簡單。結識擁有比特幣的人的一種方法是參加在 pass:[<a href="https://meetup.com" class="orm:hideurl">Meetup.com</a>] 上列出的本地比特幣聚會。
* 通過銷售產品或服務以比特幣獲得比特幣。如果你是一名程序員，可以出售你的編程技能。如果你是理髮師，可以為比特幣剪髮。
* 使用你所在城市的比特幣 ATM。比特幣 ATM 是一種接受現金並將比特幣發送到你的智能手機比特幣錢包的機器。
* 使用與你的銀行賬戶連接的比特幣貨幣交易所。((("比特幣", "貨幣交易所")))("貨幣交易所")))許多國家現在有貨幣交易所，為買賣雙方提供一個用本地貨幣交換比特幣的市場。匯率列表服務（例如 https://bitcoinaverage.com[BitcoinAverage]）通常會顯示每種貨幣的比特幣交易所列表。

[TIP]
====
比特幣相對於其他支付系統的一個優勢是，((("隱私")))如果正確使用，它為用戶提供了更高的隱私。獲取、持有和使用比特幣不需要你向第三方透露敏感和可識別的個人信息。然而，當比特幣與傳統系統接觸時，例如貨幣交易所，通常適用於國家和國際規範。為了將比特幣兌換為你的國家貨幣，通常需要提供身份證明和銀行信息。用戶應該意識到，一旦比特幣地址與身份相關聯，其他相關的比特幣交易也可能變得容易識別和跟蹤——包括早期進行的交易。這也是為什麼許多用戶選擇保持獨立於其錢包的專用交易所賬戶。
====

愛麗絲是由朋友介紹接觸到比特幣的，所以她有一個輕鬆的方法來獲取她的第一筆比特幣。接下來，我們將看看她如何從朋友喬那裡購買比特幣，以及喬如何將比特幣發送到她的((("比特幣", "獲取", startref="bitcoin-acquire")))("獲取比特幣", startref="acquire-bitcoin")))錢包中。

[[bitcoin_price]]
==== 查找比特幣的當前價格

在愛麗絲可以從喬那裡購買比特幣之前，他們必須就比特幣和美元之間的_匯率_達成一致。((("比特幣", "匯率", id="bitcoin-exchange-rate")))("匯率", id="exchange-rate")))((("比特幣的當前價格", id="current-price")))這對於那些剛接觸比特幣的人來說是一個常見問題：「誰來決定比特幣的價格？」簡短的答案是：價格由市場決定。

比特幣和大多數其他貨幣一樣，具有_浮動匯率_。這意味著比特幣的價值會根據供需關係在各個交易市場上波動。例如，比特幣的美元「價格」是根據每個市場上比特幣與美元的最新交易來計算的。因此，價格往往每秒鐘波動幾次。一個定價服務將匯集多個市場的價格並計算成交量加權平均數，以代表貨幣對（例如 BTC/USD）的廣泛市場匯率。

有數百個應用程序和網站可以提供當前的市場價格。以下是一些最受歡迎的：

https://bitcoinaverage.com[Bitcoin Average]:: 提供每種貨幣成交量加權平均值的簡單視圖的網站。
https://coincap.io[CoinCap]:: 一個列出數百種加密貨幣市值和匯率的服務，包括比特幣。
https://oreil.ly/ACieC[芝加哥商品交易所比特幣參考匯率]:: 作為投資數據提要的一部分，由 CME 提供的可用於機構和合同參考的參考匯率。

除了這些各種網站和應用程序外，一些比特幣錢包還會自動在比特幣與其他貨幣之間進行金額轉換。

[[sending_receiving]]
==== 發送和接收比特幣

愛麗絲((("比特幣", "支付", id="bitcoin-send")))("支付比特幣", id="send-bitcoin"))((("比特幣", "接收", id="bitcoin-receive")))("接收比特幣", id="receive-bitcoin")))決定購買 0.001 比特幣。在她和喬查詢匯率之後，她給了喬相應的現金，打開她的移動錢包應用程序，並選擇接收。這會顯示愛麗絲的第一個比特幣地址的 QR 碼。

然後喬在他的智能手機錢包中選擇發送並打開 QR 碼掃描器。這讓喬可以通過智能手機相機掃描條碼，而不必手動輸入愛麗絲的比特幣地址，因為地址相當長。

喬現在已經將愛麗絲的比特幣地址設置為接收方。喬輸入金額為 0.001 比特幣 (BTC)，見 <<wallet-send>>。一些錢包可能會以不同的單位顯示金額：0.001 BTC 是 1 毫比特幣 (mBTC) 或 100,000 聰 (sats)。

一些錢包還可能建議喬為此次交易輸入標籤；如果是這樣，喬輸入「愛麗絲」。幾周或幾個月後，這將幫助喬記住他為什麼發送了這 0.001 比特幣。一些錢包還可能提示喬設置手續費。根據錢包和交易的發送方式，錢包可能會要求喬輸入交易費率，或提示他選擇建議的費用（或費率）。交易費越高，交易確認的速度就越快（見 <<confirmations>>）。

.比特幣錢包發送屏幕。
image::images/mbc3_0102.png["錢包發送屏幕。圖像衍生自 Bitcoin Design Guide CC-BY"]

喬然後仔細檢查他是否輸入了正確的金額，因為他即將傳輸資金，並且錯誤將很快變得不可逆。在再次檢查地址和金額之後，他按下發送以傳輸交易。喬的移動比特幣錢包構建了一個交易，將 0.001 BTC 分配到愛麗絲提供的地址，從喬的錢包中提取資金，並使用喬的私鑰簽署交易。這告訴比特幣網絡喬已授權將價值轉移到愛麗絲的新地址。隨著交易通過點對點協議傳輸，它迅速在比特幣網絡中傳播。幾秒鐘後，網絡中大多數良好連接的節點收到交易，並第一次看到愛麗絲的地址。

與此同時，愛麗絲的錢包不斷在比特幣網絡上「聆聽」新交易，尋找任何與其包含的地址匹配的交易。幾秒鐘後，喬的錢包傳輸了交易，愛麗絲的錢包顯示它正在接收 0.001 BTC。

[[confirmations]]
.確認
****
最初，((("比特幣", "清算交易")))("清算交易")))((("交易", "清算")))("確認")))愛麗絲的地址會顯示來自喬的交易為「未確認」。這意味著交易已傳播到網絡，但尚未記錄在比特幣交易日誌中，即區塊鏈中。要確認一筆交易，它必須包含在一個區塊中，並添加到區塊鏈中，這平均每 10 分鐘發生一次。在傳統的金融術語中，這被稱為_清算_。有關比特幣交易的傳播、驗證和清算（確認）的更多詳情，請參見 <<mining>>。
****

現在愛麗絲擁有了她可以使用的 0.001 BTC。接下來的幾天中，愛麗絲通過 ATM 和交易所購買了更多的比特幣。在下一章中，我們將探討她的第一次比特幣購物，並更詳細地研究底層的交易和傳播技術。
